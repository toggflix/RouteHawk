# Security Policy

RouteHawk is a scope-safe endpoint intelligence workspace for authorized security review and learning.

## Safety Boundaries

RouteHawk must not be used as:

- an exploit scanner
- a brute-force tool
- a payload scanner
- an auth bypass automation tool
- a destructive testing tool

RouteHawk is intended for authorized, low-impact evidence collection, endpoint inventory, prioritization, and manual review guidance.

## Reporting RouteHawk Security Issues

Please report vulnerabilities in RouteHawk itself through GitHub Security Advisories if available. If advisories are not available, contact the maintainer through GitHub.

Do not report vulnerabilities found on third-party targets to this repository unless the issue is about RouteHawk itself.

## Responsible Use

Use RouteHawk only on systems you own or are explicitly authorized to test. For bug bounty programs, follow the program scope, rate limits, and rules of engagement.
